Lecture des images¶

Le code utilise la classe ImageDataGenerator de Keras pour gnrer des lots d'images augmentes partir d'un rpertoire contenant des donnes d'images pour la classification de bateaux. Voici un rsum de ce que le code fait :

Il cre une instance de ImageDataGenerator avec quelques options de transformation d'images :

  • horizontal_flip=True : permet de raliser des retournements horizontaux alatoires des images pendant l'augmentation des donnes, ce qui peut aider amliorer la robustesse du modle.
  • rescale=1./255 : rduit les valeurs de pixels des images l'intervalle [0, 1] en les divisant par 255.
  • validation_split=validation_split : spcifie la proportion des donnes utiliser pour la validation.

Ensuite, le gnrateur de donnes d'entranement (training_gen) est cr en utilisant la mthode flow_from_directory :

  • input_path : le chemin du rpertoire contenant les donnes d'images pour l'entranement et la validation.
  • target_size=(img_height, img_width) : spcifie la taille souhaite pour les images.
  • batch_size=batch_size : dfinit la taille des mini-lots d'images gnrer.
  • seed=SEED : fixe la graine alatoire pour la reproductibilit des rsultats.
  • subset='training' : indique que ce gnrateur est utilis pour les donnes d'entranement.
  • class_mode='categorical' : utilise l'encodage catgoriel des tiquettes de classe.
  • color_mode='rgb' : spcifie le mode de couleur des images en RGB.
  • shuffle=True : mlange les images chaque poque d'entranement.
  • interpolation="bicubic" : utilise l'interpolation bicubique pour redimensionner les images. L'interpolation bicubique est une mthode courante pour redimensionner les images en prservant les dtails et en attnuant les artefacts.

De mme, le gnrateur de donnes de validation (validation_gen) est cr en utilisant la mthode flow_from_directory. Les paramtres sont similaires training_gen, l'exception de subset='validation' et shuffle=False (pas de mlange des images de validation).

L'utilisation de l'interpolation bicubique pour le redimensionnement des images peut tre avantageuse car elle fournit une meilleure qualit d'interpolation par rapport d'autres mthodes plus simples, telles que l'interpolation bilinaire. Cela peut aider prserver les dtails et les contours des bateaux dans les images redimensionnes, ce qui peut tre important pour la tche de classification.

def generator_plot(gen):
    labels = gen.class_indices
    batch_size = gen.batch_size
    
    plt.figure(figsize=(16, 14))
    image, lbs = gen.next()
    num_images = image.shape[0]
    image_size = image.shape[1:3]
    num_channels = image.shape[3]
    
    print(f'Batch size: {batch_size}\nImage size: {image_size[0]} x {image_size[1]}\nNumber of channels: {num_channels}')
    
    num_rows = (batch_size + 4) // 5
    num_cols = min(batch_size, 5)
    
    for i in range(num_images):
        plt.subplot(num_rows, num_cols, i + 1)
        num_class = np.argmax(lbs[i])
        class_name = list(labels.keys())[list(labels.values()).index(num_class)]
        plt.title(class_name)
        plt.imshow(image[i])
    
    plt.tight_layout()
    plt.show()
    
    gen.reset()

Création du réseau¶

Le rseau de neurones convolutifs (CNN) dfini dans le code est un modle de classification d'images profond et complexe. Il est compos de plusieurs couches de convolution, de max pooling, de normalisation par lots (batch normalization) et de couches entirement connectes (fully connected).

Le modle commence par une couche de redimensionnement (resizing) qui adapte la taille des images d'entre 48x48 pixels, le nombre 48 a t choisi tant le PPCM(Plus Petit Commun Multiple) de 16 et 24. Ensuite, il enchane les couches de convolution avec diffrentes tailles de filtres pour extraire des caractristiques diffrentes chelles. Les couches de max pooling rduisent progressivement la taille spatiale des caractristiques extraites, tout en maintenant leur pertinence.

Des techniques de rgularisation, telles que le dropout, sont utilises pour rduire le surapprentissage. La normalisation par lots (batch normalization) est applique aprs certaines couches pour stabiliser l'apprentissage du rseau.

Aprs les couches convolutives, le modle se termine par des couches entirement connectes qui effectuent la classification. Ces couches utilisent des fonctions d'activation swish, qui sont des fonctions non linaires permettant d'introduire de la non-linarit dans le modle.

Finalement, le modle est compil avec une fonction de perte de cross-entropy catgorielle et un optimiseur Nadam, qui combine les avantages de l'algorithme Adam et de l'algorithme Nesterov Accelerated Gradient.

from keras.models import Sequential
from keras.optimizers import Adam
from keras.layers import Dense, Dropout, Flatten, Conv2D, MaxPooling2D, BatchNormalization, Resizing

init_shape=(16, 24, 3)
input_shape=(48, 48, 3)

model = Sequential([
    Resizing(48, 48, interpolation="bicubic", input_shape=init_shape),
    
    Conv2D(128, (3,3), padding='same', activation='swish', input_shape=input_shape),
    Conv2D(128, (3,3), padding='same', activation='swish', input_shape=input_shape),
    MaxPooling2D(pool_size=(2,2)),
    Dropout(0.4),
    BatchNormalization(),
    
    Conv2D(256, (3,3), padding='same', activation='swish', input_shape=input_shape),
    Conv2D(256, (3,3), padding='same', activation='swish', input_shape=input_shape),
    MaxPooling2D(pool_size=(2,2)),
    Dropout(0.5),
    BatchNormalization(),
    
    Conv2D(512, (3,3), padding='same', activation='swish', input_shape=input_shape),
    Conv2D(512, (3,3), padding='same', activation='swish', input_shape=input_shape),
    MaxPooling2D(pool_size=(2,2)),
    Dropout(0.5),
    BatchNormalization(),
    
    Conv2D(768, (3,3), padding='same', activation='swish', input_shape=input_shape),
    Conv2D(768, (3,3), padding='same', activation='swish', input_shape=input_shape),
    Dropout(0.6),
    BatchNormalization(),
    
    Conv2D(1024, (3,3), padding='same', activation='swish', input_shape=input_shape),
    Conv2D(1024, (3,3), padding='same', activation='swish', input_shape=input_shape),
    BatchNormalization(),
    Dropout(0.6),
    
    Flatten(),
    
    Dropout(0.6),
    Dense(4000, activation='swish'),
    Dense(2000, activation='swish'),
    Dense(10, activation='softmax')
])

model.compile(loss='categorical_crossentropy',
              optimizer='Nadam',
              metrics=['accuracy'])

model.build()

Le rappel EarlyStopping est utilis pour arrter l'entranement prmaturment si une mtrique de validation (dans ce cas, la prcision) cesse de s'amliorer pendant un certain nombre d'poques dfini par le paramtre "patience". Il permet d'viter un surapprentissage excessif en stoppant l'entranement lorsque la performance sur les donnes de validation ne s'amliore plus.

Le rappel ReduceLROnPlateau est utilis pour rduire le taux d'apprentissage lorsque la perte sur les donnes de validation cesse de s'amliorer pendant un certain nombre d'poques dfini par le paramtre "patience". Cela permet d'ajuster le processus d'apprentissage en rduisant progressivement le taux d'apprentissage afin d'atteindre une convergence plus prcise.
